(use-package ox-hugo
  :ensure t
  :after ox
  :config
  (setq
   ;; 设置在文件导出到content下的哪个目录，我的这个是 content/post
   org-hugo-section "post"
   ;; 设置文章的最后修改时间为导出时间
   org-hugo-auto-set-lastmod	t
   )
  ;; 设置在org文件保存的时候自动导出到org-hugo-section下
  (org-hugo-auto-export-mode))

Github Actions

我是使用了2个github仓库，一个是 github pages 用于显示博客使用的 名字.github.io，另一个是存放hugo生成的站点文件的hugo_source

使用ssh-keygen生成两组公私钥，一组用于提交hugo_source,另外一组用于hugo_source向github pages部署使用

参考博客：

  • https://blue-bird1.github.io/posts/github-action-hugo/

name: CI

# Controls when the action will run.
on:
  push:
    branches: [ master ]

  # Allows you to run this workflow manually from the Actions tab
  workflow_dispatch:

jobs:
  build-deploy:
    runs-on: ubuntu-18.04
    steps:
      - uses: actions/checkout@v1  # v2 does not have submodules option now
      with:
  	submodules: true

      - name: Setup Hugo
      uses: peaceiris/actions-hugo@v2
      with:
  	hugo-version: latest
  	# extended: true

      - name: Build
      run: hugo

      - name: Deploy
      uses: peaceiris/actions-gh-pages@v3
      with:
  	deploy_key: ${{ secrets.ACTIONS_DEPLOY_KEY }}
  	external_repository: xxx/xxx.github.io # Pages 远程仓库
  	publish_dir: "./public"
  	keep_files: false # remove existing files
  	publish_branch: master
  	commit_message: ${{ github.event.head_commit.message }}

注意哦，如果你是使用子项目的方式下载的主题，一定要使用 submodules:true ，最开始看别人的博客没有注意，直接copy使用的，然后被坑了。。。事实证明干事不能不带脑子
